# Школьный Telegram-бот (внутренний мессенджер)

Проект реализует внутренний школьный мессенджер в Telegram на `Python` + `pyTelegramBotAPI`: регистрация с подтверждением администратора, роли (ученик/преподаватель/воспитатель/остальные), уведомления, диалоги и запись в прачечную.

## Возможности

- регистрация с выбором роли и подтверждением администратором;
- отмена регистрации и повторная подача заявки;
- отправка сообщений (текст, фото, видео, документы);
- push-уведомления о новых сообщениях с кнопками действий;
- входящие уведомления `/read` и список диалогов `/dialogs`;
- ответы на сообщения и переход в диалог;
- запись в прачечную `/signup` с ограничениями по времени и по количеству машин;
- админ-раздел `/admin` (заявки, пользователи, история прачечной);
- логирование действий в отдельный файл.

## Команды

- `/start` — главное меню;
- `/register` — подать/обновить заявку на регистрацию;
- `/cancel_registration` — отменить регистрацию;
- `/send` — отправить сообщение;
- `/read` — входящие уведомления;
- `/dialogs` — диалоги;
- `/signup` — запись в прачечную;
- `/admin` — админ-раздел;
- `/pending` — быстрый доступ к заявкам.

## Установка и запуск (Windows / PowerShell)

1. Установите зависимости:

```powershell
py -m pip install -r requirements.txt
```

2. Создайте файл настроек `.env`:

```powershell
Copy-Item .env.example .env
```

3. Заполните `.env` (важно: `.env` не коммитится в Git):

```env
BOT_TOKEN=123456:ABCDEF_your_real_token_here
DATABASE_PATH=school_messenger.db
BOT_ADMIN_IDS=123456789,987654321
PAGE_SIZE=6
ACTIVITY_LOG_PATH=logs/bot_activity.log
```

4. Запуск:

```powershell
py run.py
```

Дополнительно: `py bot.py` тоже запускает бота (это совместимый враппер), но основная точка входа — `run.py`.

## Как устроен проект (файлы)

- `run.py` — точка входа: создает приложение, регистрирует обработчики и запускает polling.
- `bot.py` — совместимый враппер, который вызывает `run.py`.
- `app_context.py` — создание контекста приложения (settings/db/bot/logger).
- `config.py` — загрузка переменных окружения из `.env` (`python-dotenv`).
- `database.py` — схема `SQLite` и все операции с БД (пользователи, сообщения, прачечная).
- `access.py` — проверки доступа (зарегистрирован/одобрен/админ).
- `bot_utils.py` — общие утилиты (пагинация, клавиатуры, форматирование, общие тексты).
- `core_handlers.py` — базовые обработчики (`/start`, `noop`).
- `registration_handlers.py` — регистрация/отмена регистрации, уведомление админов о заявке.
- `messaging_handlers.py` — `/send`, `/read`, просмотр сообщения, ответы, push-уведомления.
- `dialogs_handlers.py` — `/dialogs` и навигация по диалогу.
- `laundry.py` — бизнес-логика прачечной (слоты, доступность, длительности).
- `laundry_handlers.py` — `/signup` (выбор окна, создание/отмена записи).
- `admin_handlers.py` — `/admin` (заявки, пользователи, история прачечной).
- `activity_logger.py` — запись действий в лог-файл.
- `requirements.txt` — зависимости проекта.
- `.env.example` — пример конфигурации без секретов.
- `.gitignore` — исключает из Git `.env`, базы (`*.db`) и логи (`*.log`).

## Прачечная

- запись доступна на ближайшие 2 дня;
- шаг времени: 30 минут (`xx:00` / `xx:30`), от текущего времени с округлением вверх;
- доступные окна: с `07:00` до `22:30` по Москве;
- всего машин: `3` стиральные и `2` сушильные;
- показываются только окна, где доступна хотя бы одна машина (стирка или сушка);
- ограничения на пользователя: максимум 1 будущая запись на стирку и 1 будущая запись на сушку;
- режимы стирки: `Смешанные вещи` (1.5 часа) и `Хлопок` (3 часа);
- сушка — всегда 3 часа;
- будущую запись можно отменить.

## Журнал действий

Файл журнала задается `ACTIVITY_LOG_PATH`. В него пишутся:

- отправка сообщений (кто кому, тип/превью, время);
- регистрация (подача/одобрение/отклонение/отмена);
- создание и отмена записей в прачечную.
